### 1. 画像リサイズのアルゴリズム
- **バイキュービック法 (Bicubic)**  
    最も一般的かつ高品質なアルゴリズム。周囲16個（新しい座標を囲む縦4マス × 横4マスの計16ピクセル）の情報を元に、3次関数を使って滑らかなグラデーションやエッジを計算・再現する。写真などの縮小に適する。
- **バイリニア法 (Bilinear)**  
    周囲4個（新しい座標を囲む、最も近い縦2マス × 横2マスの計4ピクセル）の距離に応じて色を平均化するアルゴリズム。バイキュービック法よりも処理が高速だが、全体的に少しぼやけた印象になる。
- **最近傍補間法 (Nearest Neighbor)**  
    最も近いピクセルの色をそのままコピーする方式。処理が非常に速い反面、色を混ぜないため境界線がギザギザになりやすい。ドット絵や機械学習用の画像処理に適する。

### 2. リサイズパッケージ
- `ImageTransformations` : 画像変形（リサイズ）用パッケージ
- `Images` : 画像入出力用のメタパッケージ
### 3. リサイズコード
- 元の画像imgを16×16ピクセルにリサイズする。
- `method=BSpline()`を使うが、画像データ（`img`）は浮動小数点（`Float`）か整数（`Int`）。グレースケール（`N0f8`）のような固定小数点はエラーになるので注意。
- バイキュービック法
	- `Cubic()` : 周囲16個のピクセルを使い、3次関数（曲線）の数式に沿って滑らかに色を補完した値を返す。
	- `Line()` : 境界（画像の端）の処理方法として、線形補外を適用
	- `OnGrid()` : ピクセルの中心がグリッド上にある（一般的なデジタル画像）として計算
例：
 ```
img_resized = imresize(img, (16, 16), method=BSpline(Cubic(Line(OnGrid()))))
 ```
- バイリニア法
	- `Linear()` : 周囲4個のピクセルとの距離を測り、直線的に（比例計算で）色を混ぜ合わせた値を返す。
例：
```
img_resized = imresize(img, (16, 16), method=BSpline(Linear()))
```
- 最近傍補完法
	- `Constant()` : 座標を四捨五入して常に一定の最も近いピクセル値を返す。
例：
```
img_resized = imresize(img, (16, 16), method=BSpline(Constant()))
```
