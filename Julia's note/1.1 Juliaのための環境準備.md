### 1. 準備すべき環境
- **Juliaup (インストーラー)** Windowsストアか公式から入れる。Juliaのバージョン管理（1.10から1.11への切り替えなど）がコマンド一つで完結。
- **VS Code + Julia Extension** これが「実質的な統合開発環境」になる。Dockerコンテナ不要。素早くデバッグや変数の確認が可能。
- **Pkg（Julia標準機能）** プロジェクトごとに `Project.toml` というファイルが作られ、どのバージョンのパッケージを使ったか記録される。これがあれば将来別のPCに移る時も、Dockerイメージを使わず同じ環境を再現できる。

### 2. Dockerを使わない理由
- **GPUアクセラレーションの複雑化** JuliaからGPUを叩く際、ネイティブ環境ならライブラリ（CUDA.jl）が自動で必要なツールを揃えてくれる。Docker経由だと「NVIDIA Container Toolkit」の設定や、ホストOSとのドライバー整合性など、機械学習を始める前に「インフラ構築」で時間を取られる。
- **Juliaのパッケージ管理が優秀すぎる** PythonでDockerが多用されるのは、ライブラリの依存関係が壊れやすい（Condaやpipの混在など）から。Juliaの標準機能である `Pkg` は、ディレクトリごとに完全に独立した環境を数秒で作れるため、Dockerを使わなくても「環境が汚れる」心配がほとんどない。
- **対話型開発（REPL/Jupyter）との相性** Juliaの強みは、コードを1行ずつ実行しながら量子回路の挙動を確認するような「試行錯誤」にある。Docker越しだと、グラフの描画（Plots.jlなど）やエディタ（VS Code）との連携に、ポート開放などの手間が発生する。

※**REPL** (Read-Eval-Print Loop) ：入力・評価・出力のループ
