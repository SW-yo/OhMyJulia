1. ドット演算子 `.` (ブロードキャスト)
	- 関数の後ろに `.` をつけるだけで「配列の全要素に対して実行せよ」という命令になる。
例：
```
x = [1, 2, 3]

# for文を使わずに、全要素を2倍にする
y = x .* 2  

# 全要素に sin を計算する
z = sin.(x)
```
- 「基礎」として重要性
	- **直感的:** 行列 `A` の全要素に `10` を足したいとき、`A .+ 10` と書くだけで済む。
	- **爆速:** Julia の内部で最適化されるため、自分で `for` 文を書くよりも速く、メモリ消費も抑えられる。
	- **GPU との相性:** このドット演算子はそのままGPUへの命令に変換できる。
例：
```
A = [1 2; 3 4]

# パターン1 (行列の掛け算)
B1 = A * A

# パターン2 (要素ごとの掛け算)
B2 = A .* A
```
- 引数のブロードキャスト
例：
```
# sin()の引数にベクトルを使う
# sin([1, 2, 3])はエラーになる
sin.([1, 2, 3])

# 行列も同様に扱える
sin.([1 2 3; 4 5 6])
```
- 行列要素ごとのブロードキャスト（要素積/アダマール積）
例：
```
a = [1, 2, 3]
b = [4, 5, 6]
c_matrix-product = a * b    # 行列積
c_hadamard-product = a .* b # 要素積
```
- 定義した関数のブロードキャスト
	- Juliaのブロードキャスト `.` は、デフォルトでは「配列の中身（要素）」を一つずつ取り出そうとする。
	- `sum_of_sqrt.((a, b))` とタプルを使って代入するとブロードキャスト対象は各配列ごとになる。
	- **「ひとかたまりとして扱うために包む」** というテクニックは、Juliaのブロードキャストを使いこなす上で非常に便利。
	- ブロードキャスト以外に`map`関数や内包表記でも同じ結果が得られる。
例：
```
# 関数
function sum_of_sqrt(arr)
    r = sqrt(arr[1]) + sqrt(arr[2])
    return r
end

a = [4; 9]
b =[16,  25]
# c_test1 = sum_of_sqrt(a, b)   これはエラー
c_test2 = sum_of_sqrt.(a, b)    # これはa,bの第1要素、第2要素ごとの組み合わせ
# c_test3 = sum_of_sqrt.(a b)   # 空白区切りは文法的にエラー
c_test4 = sum_of_sqrt.((a, b))   # タプルで送るとa,bそれぞれを関数処理

# 同様の結果が得られる書き方
map(sum_of_sqrt, (a, b))    # map関数
[sum_of_sqrt(x) for x in (a, b)]    # 内包表記
```
