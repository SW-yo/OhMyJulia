### 1. `for`での並列処理
- `Threads.@threads` : `for` ループ専用のマクロ
- `for`文などで内容が競合しない処理を並列化できる。
例：
```
Threads.@threads for i in 1:20
        println("i=$i, thread=$(Threads.threadid())")
        # ホールドアウト法              
        rate_holdout[i] = kNN_holdout(records_digits, 650, 650, k)[2]
        #ジャックナイフ法
        rate_jackknife[i] = kNN_jackknife(records_digits, 650, k)[2]
    end
```
- そのまま動くケース：
	- 各スレッドが別のインデックスに書き込む（今回のように `rate_holdout[i]`）
	- 読み取りのみ（`records_digits` の参照）
- 競合して壊れるケース：
	- 複数スレッドが同じ変数に `+=` や `push!` する
	- グローバル変数を書き換える
- 競合しないようにスレッド内ではインデックスを付けて結果を保持し、計算終了後に集計する。

### 2. その他の並列処理
- **`Threads.@spawn`** : 任意のコードをスレッドで非同期実行
- **`@async`** : 非同期タスク（I/O待ちに向いている）
- **`pmap`** : 関数を並列マップ（`Distributed.jl`）

```julia
# @spawnの例
t1 = Threads.@spawn kNN_holdout(...)
t2 = Threads.@spawn kNN_jackknife(...)
result1 = fetch(t1)
result2 = fetch(t2)
```
