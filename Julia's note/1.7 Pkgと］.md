### 1. Pkgの特長
- ]でアクティベイトするとそのPkgの名前でそのプロジェクト用の仮想環境が自動的に立ち上がる
- アクティベイト：`activate .` （ドットは「今の場所」の意味）と打つと、Juliaはそのフォルダに `Project.toml` があるか見に行く。    
- 自動生成：もしファイルがなければ空の `Project.toml`を自動で作ってくれる。
- 名前の紐付け：REPLのプロンプト（左側の文字）が `(Julia_Study) pkg>` のようにフォルダ名に変わる。

|**機能**|**Python (uv / venv)**|**Julia (Pkg)**|
|---|---|---|
|**環境の有効化**|`.venv` 作成 / `activate`|`]` キーから `activate .`|
|**ライブラリ一覧**|`pyproject.toml`|`Project.toml`|
|**詳細な固定情報**|`uv.lock`|`Manifest.toml`|
|**実体の保存場所**|各プロジェクトの `.venv` 内|`~/.julia/packages` (一括管理)|

### 2. モードの切り替え
- **通常モード (`julia>` )：** そのままプログラムを書いたり計算したりできる場所
- **pkgモード (`(@v1.11) pkg>` ):** `]` を押す。ライブラリをインストールしたり、仮想環境を操作したりする「管理専用」の場所
- **戻り方** `Backspace` キー（または `Ctrl + C`）

### 3. インストール（add）とインポート（using）
- Pythonでいうところの`pip`と`import`
- インポートはpkgモードで`add [library]`
- 使うときはコード内かコマンドラインで`using [library]`（こっちが基本動作）
- 名前を付けたい時は`import [library] as [name]
- Pythonの`numpy.dot()` のようにメソッドを指定する時、モジュール名から指定する必要がない。

| **やりたいこと**           | **Python (NumPyなど)**          | **Julia (Standard)**          |
| -------------------- | ----------------------------- | ----------------------------- |
| **中身を全部そのまま使いたい**    | `from math import *`          | **`using Statistics`**        |
| **特定の機能だけ使いたい**      | `from math import sqrt`       | **`using Statistics: mean`**  |
| **名前を短くしたい (Alias)** | `import pandas as pd`         | **`import DataFrames as df`** |
| **モジュール名をつけて呼びたい**   | `import math` / `math.sqrt()` | **`import LinearAlgebra`**    |

### 4. 呼び方
- パッケージ：ライブラリ相当
- モジュール：メソッド相当
- 構造体（Struct）：クラス相当

| **特徴**     | **Python (クラス)**            | **Julia (構造体 + 関数)**              |
| ---------- | --------------------------- | --------------------------------- |
| **データの定義** | `class Dog:` (データと関数が同居)    | **`struct Dog`** (データのみ)          |
| **命令の定義**  | `def bark(self):` (クラスに属する) | **`bark(d::Dog) = ...`** (独立した関数) |
| **呼び出し方**  | `pochi.bark()` (ドットで繋ぐ)     | **`bark(pochi)`** (関数に渡す)         |

### 5. オブジェクト指向
- 3大要素

| **要素**       | **Python (クラス)**          | **Julia (型システム)**                |
| ------------ | ------------------------- | -------------------------------- |
| **カプセル化**    | クラスの中にデータとメソッドを閉じ込める      | **モジュール**で範囲を区切り、**構造体**でデータを固める |
| **継承**       | クラスを引き継ぐ（親のメソッドも継承）       | **型階層（抽象型）**を引き継ぐ（振る舞いは関数で定義）    |
| **ポリモーフィズム** | `obj.method()` で中身に合わせて動く | **多重ディスパッチ**（引数の型で最適な関数を選ぶ）      |
- 設計思想

|**概念**|**一般的なオブジェクト指向**|**Juliaの設計思想**|
|---|---|---|
|**主役**|オブジェクト（物）|**関数（動作）**|
|**データの持ち方**|クラス（メソッドと一体）|**構造体（データのみ）**|
|**共通化の方法**|クラス継承|**抽象型（Abstract Type）の階層**|
|**実行の仕組み**|メッセージパッシング|**多重ディスパッチ**|
