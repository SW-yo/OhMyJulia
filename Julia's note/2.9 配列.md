### 1. 基本的な書き方
- `[要素1, 要素2, …, 要素N]`
例：
```
x = [10, "march", 3.14]
```
- 要素は縦ベクトル状に格納される。
- 縦ベクトルはカンマ`,`ではなくセミコロン`;`で区切ることで行列の記載時の混乱を避ける。


### 2. 辞書
- 宣言は`Dict()`
例：
```
d = Dict()
d["xyz"] = 123    # 数字もOK
d["name"] = "frieren"    # 文字列もOK
d[]
```

- 単に `Dict()` と書くと「何でも入る辞書（`Any`型）」になってしまう。
- あらかじめ型を指定するのがJulia流
- 書き方： `Dict{キーの型, 値の型}()`
例：
```
# キーは文字列(String)、値は整数(Int)限定の辞書
d = Dict{String, Int}()

d["orange"] = 150  # OK
d["melon"] = "expensive" # エラー（値が整数じゃないため）
```
- 要素の一覧表示
例：
```
for (key, value) in d
    println(key, ":", value)
end
```
- 基本操作

| 操作             | コード例                  | 備考                                       |
| :------------- | :-------------------- | :--------------------------------------- |
| **値の参照**       | `d["apple"]`          | キーが存在しない場合はエラーになる                        |
| **値の追加・更新**    | `d["grape"] = 300`    | 新規なら追加、既存なら値を上書き                         |
| **キーの確認**      | `haskey(d, "apple")`  | 存在すれば `true`、なければ `false` を返す            |
| **デフォルト値付き参照** | `get(d, "cherry", 0)` | キーがない場合にエラーにせず、指定した値（例では `0`）を返す         |
| **要素の削除**      | `delete!(d, "apple")` | 指定したキーのペアを削除。`!` は「破壊的（元のデータを変更する）」という意味 |
| **全キーの取得**     | `keys(d)`             | キーの一覧（イテレータ）を返す                          |
| **全値の取得**      | `values(d)`           | 値の一覧（イテレータ）を返す                           |

### 3. タプル
- `()`で囲んで作成
例：
```
# 基本的なタプル
t = (1, 2, "Julia")

# 要素が1つだけの場合はカンマが必要
t_single = (1,)
```
- 多重代入（アンパック）
例：
```
x, y = (10, 20)
# x = 10, y = 20 になる
```
- 関数の戻り値をタプルで返す
例：
```
function get_pos()
    return (100, 200)
end

x, y = get_pos()
```
- 名前のついたタプル（辞書より軽量で高速）
例：
```
# 作成
person = (name="Taro", age=25)

# 参照（ドットでアクセスできる！）
println(person.name) # "Taro"
```

### 4. 使い分け
- 「要素の変更可否（ミュータビリティ）」「要素の型」「次元数」の3点で行う。用途に応じて最も高速・安全なデータ構造を選択する。
- 
1. タプル (Tuple)
	- **表記法**: 丸括弧 `(1, 2, "A")`
	- **特徴**: **変更不可（イミュータブル）**、要素数は固定。異なる型のデータを混在できる。
	- **使い分け**: 関数の複数戻り値、変更されては困る設定値、辞書のキー、要素数が非常に少ないペア（座標など）で使用。メモリ消費が少なく、非常に高速。

2. アレイ（配列 / Array）
	- **表記法**: 角括弧 `[1, 2, 3]` または `Array{T}(undef, 2, 2)`
	- **特徴**: **変更可能（ミュータブル）**、多次元（行列やテンソル）の表現が可能。ただし、原則として要素はすべて「同じ型」。
	- **使い分け**: 多次元データや、サイズを後から変更する（要素を追加・削除する）動的なリストデータを扱う場合。

3. ベクトル (Vector)
	- **表記法**: 角括弧 `[1, 2, 3]`
	- **特徴**: アレイの一種（1次元配列の別名）。線形代数の列ベクトルとして扱われ、最も頻繁に使用されます。
	- **使い分け**: 数学的なベクトルの計算、データの順序付きリストとして扱う場合。

4. 辞書 (Dict)
	- **表記法**: `Dict("key1" => 1, "key2" => 2)`
	- **特徴**: **変更可能（ミュータブル）**。インデックス（1番目、2番目…）ではなく、**「キー（Key）と値（Value）のペア」**でデータを管理する。
	- **使い分け**: データの順番に意味がなく、特定の名前や識別子を使ってピンポイントで値を検索・更新したい場合。

- 使い分け
	- **変更しない ＆ 異なる型をまとめる** ➡ **タプル**
	- **要素を変更する ＆ 1次元のリスト** ➡ **ベクトル** (`[1, 2, 3]`)
	- **要素を変更する ＆ 2次元以上の行列** ➡ **多次元アレイ** (`[1 2; 3 4]`)
	- **「名前」と「値」をセットで管理する ＆ 検索を高速化する** ➡ **辞書 (Dict)**
### 4. collect()
- `collect()` : `Range`や`Iterator`を`Array`に変換する関数
- 実用上は下記で使う（使用場面は少ない）。
	1. 関数が`Array`しか受け付けない時
	2. 要素を書き換えたい時（例えば`Range`はimmutable）
例
```
x = -3π:0.1:3π   # StepRangeReal（遅延評価）
collect(x)         # Vector{Float64}に変換
```
- `Range` : 実際には数列を保持していなく、`start`/`step`/`stop`だけ持つ軽量オブジェクト。`collect`で初めて全要素がメモリに展開される。
- `Iterator` : 「次の要素を1つずつ取り出せる」オブジェクトの総称。全要素をメモリに持たず、要求されたときに1つずつ生成。
例：
```
# これらは全部イテレータ
-3π:0.1:3π          # Range
enumerate(v)         # インデックス付き
zip(a, b)            # 複数配列を束ねる
filter(f, v)         # 条件でフィルタ
keys(dict)           # Dictのキー
```
- `for`文や`lines!`等の多くの関数はRangeをそのまま受け付けるので、実際には`collect`不要なケースが多い。
### 5. 宣言
- 宣言方法

| 型          | 構文                          | 用途        |
| ---------- | --------------------------- | --------- |
| Vector     | `Float64[]`                 | 1型の可変長リスト |
| タプルのVector | `Tuple{Float64, Float64}[]` | 2要素ペアのリスト |
| Dict       | `Dict{String, Float64}()`   | キー→値マッピング |

- 要素の型と次元を明示（基本）
```
# 32ビット整数型の1次元配列（ベクトル）の宣言
a = Array{Int32, 1}() # 1次元の場合のみサイズ指定不要

# 64ビット浮動小数点数型の2次元配列（行列）の宣言
b = Array{Float64, 2}(undef, m, n) # m×nのマトリクス、undefは必要

```
- よく使うエイリアス
```
# Vector を使う例
v = Vector{Int64}([10, 20, 30])

# Matrix を使う例
m = Matrix{Float64}(undef, 2, 2)

```

**法則：**
- 配列系 → `型[]`
- Dict系 → `Dict{K, V}()`（`[]`記法がない）

### 6. 追加・削除操作
| 操作       | 関数                                     | 備考       |
| -------- | -------------------------------------- | -------- |
| 1要素追加    | `push!(a::Array, x) `                  | 末尾に1個    |
| 複数追加     | `append!(a::Array, v::AbstractVector)` | 末尾に配列を結合 |
| インデックス削除 | `deleteat!(a::Array, i)`               | 位置指定     |
| 条件削除     | `filter!(f::Function, a::Array)`       | 破壊的      |
| 差集合      | `setdiff(A, B)`                        | 非破壊的     |
- 型注釈として`push!(a::Array, x) -> Array`や`setdiff(A, B) -> Vector{T}`を付けることで戻り値の型を明示したり、型安定性の向上を図ることができる（ない場合は`Any`）
